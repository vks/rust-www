Rust 커뮤니티

Rust 프로그래밍 언어에는 여러 특장점이 있지만, 그 가운데 가장 뛰어난 점은 Rust를 쓰는 것이 보람된 경험이 될 수 있도록 함께 하는 커뮤니티입니다.

저희는 성별, 성적 지향, 장애 여부, 민족, 종교를 포함한 개인적 특징과 상관 없이 누구에게나 친절하고, 안전하며 따뜻한 환경을 만들려 하고 있습니다. 저희의 행동 규칙은 모든 공식 Rust 포럼에서의 행동에 기준을 정합니다.

만약 커뮤니티 구성원에게 괴롭힘이나 불편함을 당했었거나 당하고 있다면, Rust 중재 팀 누구에게나 즉시 연락해 주십시오. 당신이 자주 기여하는 사람이든 새로 오는 사람이든, 저희는 커뮤니티를 당신에게 안전한 장소로 만드는 데 주의를 기울일 것입니다.

시작하기

Rust를 시작하는 사람들에게 가장 중요한 커뮤니티 자원은 다음이 있습니다:

  • #rust-beginners는 어떤 수준의 질문이든 답변하길 좋아하는 IRC 채널입니다.
  • 사용자 포럼은 Rust에 대한 모든 것에 대해 논하는 장소입니다.

질문·답변 사이트 Stack Overflow에서도 도움을 받을 수 있습니다.

소식

〈이번 주 Rust에서는(This Week in Rust)〉에서는 Rust 언어와 라이브러리에 대한 최신 소식과, 앞으로의 행사 및 주 단위 변경점들을 모읍니다. Rust 블로그는 Rust 팀이 중요한 발전을 소개하는 장소입니다. 그리고 Rust에서 일어나는 거의 모든 일들은 비공식 서브레딧 /r/rust에서도 함께 의논됩니다.

저희는 트위터 계정도 운영합니다.

IRC 채널

Rust 커뮤니티는 Mozilla의 IRC 네트워크, irc.mozilla.org에서 친근하고 북적이는 IRC 채널 여러 개를 운영하고 있습니다.

#rust 채널은 Rust에 대해 일반적인 이야기를 나누는 장소로서 도움을 요청하기에 좋습니다. 채널 사람들은 Rust에 대한 어느 질문에나 답하고자 하고, 보통 빠르게 답변을 받을 수 있습니다.

Rust 개발자들은 #rust-internals에 모입니다. 이 채널은 Rust 언어 자체를 해킹하는 데 대한 실시간 논의를 위한 것이며, Rust에 기여하는 데 대한 질문을 하는 데도 쓰입니다.

주요 채널

  • #rust는 Rust에 대해서라면 아무거나 다룹니다.
  • #rust-beginners는 Rust를 처음 접하는 사람들을 위한 곳으로 #rust보다 덜 북적입니다.
  • #rust-internals에서는 Rust 프로젝트 자체의 개발에 대해 논의합니다.

  • #rust-docs는 문서화 팀의 보금자리입니다.

  • #rust-lang은 언어 팀의 보금자리입니다.

  • #rust-mods는 중재 팀의 IRC 채널입니다.
  • #rust-style는 스타일 팀의 보금자리입니다.

주제별 채널

이 채널들은 더 넓은 범위의 Rust 커뮤니티를 위한 채널들로 중재 팀이 중재하지는 않습니다.

  • #rust-bots에서는 여러 봇들이 Rust에 대한 알림을 전달합니다.
  • #rust-crypto는 Rust로 암호학을 하는 데 대해 다룹니다.
  • #rust-embedded는 Rust로 임베디드 개발을 하는 사람들이 모입니다.
  • #rust-gamedev에는 Rust로 게임 개발을 하는 사람들이 모입니다.
  • #rust-geo는 Rust로 작성된 지리공간(geospatial) 도구인 GeoRust를 다룹니다.
  • #rust-machine-learning은 Rust로 기계학습을 하는 사람들이 모입니다.
  • #rust-networking에는 Rust로 컴퓨터 네트워킹을 하는 사람들이 모입니다.
  • #rust-osdev에는 Rust로 운영체제 개발을 하는 사람들이 모입니다.
  • #rust-webdev에는 Rust로 웹 개발을 하는 사람들이 모입니다.
  • #hyper는 잘 알려진 Rust HTTP 라이브러리인 Hyper를 다룹니다.
  • #iron은 확장 가능한 Rust 웹 프레임워크인 Iron을 다룹니다.
  • #nom은 Rust 파서 컴비네이터 라이브러리인 Nom에 대해 다룹니다.
  • #redox는 Rust로 작성된 운영체제인 Redox에 대해 다룹니다.
  • #rocket은 잘 알려진 Rust HTTP 라이브러리인 Rocket을 다룹니다.
  • #serde은 유명한 Rust 직렬화 라이브러리인 Serde에 대해 다룹니다.
  • #servo는 Rust로 작성된 브라우저 엔진인 Servo에 대해 다룹니다.
  • #xi는 Rust로 작성된 텍스트 편집기인 Xi를 다룹니다.

포럼

비동기적인 논의를 위해서 두 개의 포럼이 있습니다:

  • 사용자(users) 포럼은 질문을 하고, 코드 조각을 올리거나, Rust 프로젝트에 대해 얘기하거나 하는 장소입니다.

  • 내부(internals) 포럼은 Rust 자체(Cargo, 표준 라이브러리 및 기타 핵심 인프라를 포함합니다)에 대한 설계와 구현을 논의하는 장소입니다.

유튜브 채널

Rust 유튜브 채널에는 Rust 커뮤니티 사람들이 사용자 그룹과 컨퍼런스에서 발표한 것이 올라 옵니다.

사용자 그룹과 모임

전 세계 35개 이상의 나라에 90개 이상의 Rust 사용자 그룹이 있습니다. Rust 사용자들은 주기적으로 Rust 사용자 그룹에 모입니다. 사용자 그룹은 커뮤니티를 접하고, 비슷한 흥미를 가진 다른 사람들과 함께 배우며 친교를 나누기에 좋은 곳입니다. 모임은 보통 달마다 있으며 격식 없이 누구나 참여할 수 있습니다.

전 세계의 Rust 행사들을 모으는 달력이 있습니다. 여기에 새 행사를 추가하려면 커뮤니티 팀에게 연락해 주세요.

Rust 팀

Rust는 커뮤니티가 주도하는 개발 프로세스를 따르며, 대부분의 결정은 다음 팀들의 인도 하에 열린 토론과 합의에 따라 이루어집니다:

  • 코어 팀은 설계 및 개발 프로세스를 이끌고, 새 기능의 추가를 감독하며, (매우 드뭅니다만) 합의가 이루어지지 않을 때 결정을 내리는 역할을 합니다.

  • 언어 팀은 새 언어 기능을 설계하는 역할을 합니다.

  • 라이브러리 팀은 Rust 표준 라이브러리, rust-lang 크레이트 및 코딩 규약을 다루는 역할을 합니다.

  • 컴파일러 팀은 컴파일러 내부와 최적화를 다루는 역할을 합니다.

  • 개발 도구 팀은 Rust 코드를 다루는 도구들의 전반적인 방향을 정합니다.

  • Cargo 팀은 Cargo의 설계와 구현을 담당합니다.

  • 인프라 팀은 프로젝트 자체를 지지하는 인프라들, 이를테면 지속적 통합(CI), 릴리스 생성, 봇 및 수치 등을 관리합니다.

  • 커뮤니티 팀은 행사를 조율하고, 외부 연락, 상용 사용자들, 교육 자료들 및 프로젝트 노출을 관리하는 역할을 합니다. 커뮤니티 팀은 또한 Rust 관련해서 누구에게 연락해야 할지 모를 때 적절한 관계자를 연결해 줄 수도 있습니다.

  • 중재 팀은 행동 규칙이 잘 지켜지도록 돕는 역할을 합니다.

  • 문서 팀은 Rust가 멋진 문서를 가지도록 하는 역할을 합니다.

공식 팀 구성원과는 별개로, 대부분의 팀에는 해당 분야에 대해 잘 알고 코드를 확인할 수 있는 더 많은 리뷰어들이 있습니다. 이들 중 한 팀에 참여하고 싶다면, 부담 없이 팀 리더나 다른 구성원에게 요청해서 시작할 수 있도록 도움을 받으세요.

Rust 개발

Rust에는 2,000 명의 서로 다른 기여자가 참여해 왔고, 이 숫자는 매주 늘고 있습니다. 여러분도 이 목록에 들어 갔으면 정말 좋겠네요!

앞에서 설명했듯 Rust 내부(internals) 포럼은 Rust의 설계와 구현을 다룹니다. GitHub에서도 많은 논의가 이루어집니다:

  • 메인 저장소와 이슈 트래커는 구현 작업의 최전선입니다. 저희 리뷰어들은 최대한 친근하려고 노력하며, 새 기여자들을 돕고자 합니다. 주저하지 말고 풀 요청(pull request)을 하세요!

  • RFC 저장소는 저희의 의견 요청(Request for Comment) 과정을 추적합니다. 이 과정은 Rust 커뮤니티와 팀이 언어, 공식 라이브러리와 도구에 대해 제안된 새 기능들에 대한 합의에 도달하는 주된 경로입니다.
